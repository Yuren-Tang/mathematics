# Programme B3–B8 finite-certificate assurance ledger

## 1. Assurance classes

Every finite result used in the B3–B8 corpus has one of the following statuses.

| Class | Meaning |
|---|---|
| `F-PROVED` | the finite statement has an independent mathematical proof in the active theorem spine; computation is corroboration only |
| `F-CERT-PUBLIC` | an explicit public object, table, or dependency-free enumeration description is available in principle for checking; no rerun is asserted by curation |
| `F-CERT-PRIVATE` | exact statement and refs are public, but supporting code/notebooks remain private or on recovery branches |
| `F-CENSUS` | a named finite laboratory count used for calibration, not as a premise of a universal theorem |
| `CODE-PARTIAL` | recovered combined verification is incomplete or contains an unavailable historical assertion; no theorem depends solely on it |
| `AFFECTED` | the finite data may remain correct, but a former downstream inference has been invalidated or narrowed by a correction |

Curator inclusion does not upgrade any class.

## 6. `CODE-PARTIAL` boundary

The recovered combined verifier was incomplete and one historical assertion remains unavailable. Reconstructed modules may supply evidence, but the old combined verifier is not a public green end-to-end certificate.

No theorem may rely solely on:

- hidden notebook state;
- an unavailable assertion;
- an unexplained raw count;
- a witness without edgewise validation;
- a finite table whose consuming theorem changed scope under B1–B7 corrections.

## 7. Minimum metadata

A finite result promoted in the active corpus should record:

1. exact source commit and path;
2. exact finite object and labelling convention;
3. graph/flow/lift identifier or canonical edge list;
4. code or dependency-free enumeration description when available;
5. raw output and group action convention;
6. explicit witness for existential or counterexample claims;
7. consuming theorem;
8. assurance class;
9. every scope correction affecting its interpretation.

## 8. Boundary

B8 is assurance normalization. It is not independent mathematical review, formal verification, public-code reproducibility, manuscript approval, publication, release, arXiv, or DOI status.

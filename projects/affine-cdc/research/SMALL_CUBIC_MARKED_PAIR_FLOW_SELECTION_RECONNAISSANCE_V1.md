## 1. Root-flow encoding

Represent the ten roots of `E_5` as the two-subsets of `[5]`.

For two marked edge values `p,q`, write:

- `A` when `p=q`;
- `B` when `p,q` are distinct and intersect;
- `C` when `p,q` are disjoint.

Equivalently, if a root flow is represented by five even subgraphs and `S(e) subset [5]` is the two-element support of edge `e`, then

\[
|S(e)\cap S(f)|=
\begin{cases}
2,&A,\\
1,&B,\\
0,&C.
\end{cases}
\]

A root flow is exactly a multiset of five cycle-space vectors such that every graph edge occurs in exactly two of them. Quotienting by support permutation allows the five vectors to be enumerated as a multiset.

## 4. Minimal counterexample to naked selection

In `K_4`, every pair of opposite edges has possible intersection set

\[
\boxed{\{0,2\}}.
\]

Thus such a pair may be `A` or `C` but is never `B` in any root flow.

No pair of opposite edges in `K_4` is a two-edge cut: `K_4` is three-edge-connected.

Therefore:

### Theorem 4.1 — non-cut selection is false

There exists a three-edge-connected simple cubic graph and a pair of edges which admits root flows but admits no root flow placing the pair in the `B` orbit.

Consequently the proposed statement

\[
\text{not a two-edge cut}
\Longrightarrow
\exists\text{ a root flow with marked pair in }B
\]

is false.

## 5. Adjacent pairs are rigidly `B`

Every adjacent edge pair in every enumerated graph has possible intersection set

\[
\boxed{\{1\}}.
\]

This is not merely computational.

At a cubic vertex, three incident roots `p,q,r` satisfy

\[
p+q+r=0.
\]

Three weight-two vectors with zero sum form one `K_5` root triangle. Hence every two incident roots are distinct and intersect.

### Theorem 5.1 — adjacent-edge rigidity

In every root-valued `E_5` flow on every cubic graph, the values on any two adjacent edges lie in the `B` orbit.

This does not solve the canonical weld problem because the two reconnecting weld edges are generally not adjacent after cancellation.

## 6. Why naked `B`-existence is vacuous for the actual weld

The canonical `C6` star before cancellation is already root-valued. Its equal-face cancellation reconnects:

- the two `z` incidences into one edge `e_z` of value `z`;
- the two `w` incidences into one edge `e_w` of value `w`;

where

\[
z\ne w,
\qquad
z+w\in R_5.
\]

Therefore the inherited lower-order flow already satisfies

\[
\boxed{(\lambda(e_z),\lambda(e_w))=(z,w)\in B.}
\]

So the assertion

\[
\exists\text{ some root flow on the cancelled graph with }(e_z,e_w)\in B
\]

requires no theorem at all: the constructing flow is such a witness.

The reason this witness does not close contextual return is that inverse welding it simply recovers the canonical star state from which the cancellation was made. It supplies no proved progress toward the original history context.

## 7. Corrected Repair C target

A useful flow-selection theorem must impose an additional progress condition. Schematically it must select a lower-order root flow `rho` such that:

1. the weld pair remains in `B`, so inverse insertion is root-valued;
2. the reinserted order-`N` root state does not merely return to the same recurrent contextual component;
3. it either consumes a strict original-history prefix, reaches an original-cap terminal, or decreases an independently proved macro rank.

Thus the actual target is not

\[
\exists\rho:(e_z,e_w)\in B,
\]

but something of the form

\[
\boxed{
\exists\rho:
(e_z,e_w)\in B
\quad\text{and}\quad
\operatorname{Progress}(\rho).
}
\]

The meaning of `Progress` is precisely the unresolved global contextual datum.
